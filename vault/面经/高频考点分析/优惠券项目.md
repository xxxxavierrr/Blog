1. 介绍一下技术栈

该项目主要采用了 Spring Boot 作为基础框架，结合 Spring Cloud 和 Spring Cloud Alibaba 实现微服务架构。数据库方面使用了 MySQL，并通过 ShardingSphere 进行分库分表操作。缓存使用 Redis，消息队列采用 RocketMQ，定时任务使用 XXL - Job。

2. SpringBoot用的版本号 最低支持JDK版本号 对应的Spring大版本号多少

Spring Boot 版本号：3.3.4

最低支持 JDK 版本号：Spring Boot 3.3.x 最低支持 JDK 17

对应的 Spring 大版本号：Spring Boot 3.3.x 对应 Spring Framework 6.x

3. 高并发怎么不超卖（3）

Lua 脚本保证原子性：使用 Lua 脚本在 Redis 中执行库存检查和扣减操作，确保这两个操作在一个原子操作中完成，避免并发情况下的竞态条件。

数据库乐观锁：在数据库层面使用乐观锁机制，例如在优惠券模板表中添加版本号字段，每次更新库存时检查版本号是否与查询时一致，若一致则更新并更新版本号，若不一致则表示数据已被其他线程修改，需要重新处理。

4. Lua脚本+Redis怎么保证原子性（3）

Redis 是单线程执行命令的，Lua 脚本在 Redis 中会作为一个整体被执行，在脚本执行过程中不会被其他命令插入或打断，从而保证了脚本中多个操作的原子性。

注意：Redis原子性和MySQL事务原子性不是一个意思。

5. Lua脚本怎么实现的

```lua
local key = KEYS[1] -- Redis Key
local userSetKey = KEYS[2] -- 用户领券 Set 的 Key
local userIdAndRowNum = ARGV[1] -- 用户 ID 和 Excel 所在行数
-- 获取库存
local stock = tonumber(redis.call('HGET', key, 'stock'))
-- 检查库存是否大于0
if stock == nil or stock <= 0 then
    return false
end
-- 自减库存
redis.call('HINCRBY', key, 'stock', -1)
-- 添加用户到领券集合
redis.call('SADD', userSetKey, userIdAndRowNum)
return true
```

6. 除了Lua还有什么能实现原子性

Redis 事务：使用MULTI/EXEC等命令将多个命令打包执行，但 Redis 事务不支持回滚。

分布式锁：使用 Redis 或 ZooKeeper 实现分布式锁，保证同一时间只有一个线程可以访问共享资源。

7. Redis在项目里还有什么用处

缓存预热：在创建优惠券模板时，将数据库中的记录放入 Redis 缓存，提高数据访问速度。

库存扣减：在高并发场景下，先在 Redis 中进行库存预减操作，避免直接访问数据库带来的性能问题。

用户领券记录：使用 Redis 的 Set 或 ZSet 数据结构记录用户的领券信息，方便快速查询和统计。

分布式锁：使用 Redis 实现分布式锁，保证多个服务之间的并发控制。

延迟队列：使用Redis的ZSet实现延迟队列

8. Redis延迟队列兜底机制怎么实现的

当用户预约的抢购优惠券提醒任务加入线程池后，向Redis堵塞队列添加任务Key，20秒后检查任务Key是否已经执行后删除，若无，则发送给MQ再次进行提醒操作。
